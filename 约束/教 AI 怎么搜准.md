# 教 AI 怎么搜准

## 1. 目的

本文是给 AI 自己执行的检索约束。

目标只有一个：

**当用户要求网络搜索、查资料、找官方依据、搜最佳实践时，AI 必须用更高精度的检索提示词与检索步骤，搜到真正相关、可引用、可落地的资料。**

本文面向 AI，不面向用户教学。

## 3. AI 必须遵守的总原则

### 3.1 先收窄，再搜索

AI 在搜索前，先把用户问题压缩成明确检索目标。

禁止直接拿宽泛原句去搜：

- `查一下 DDD`
- `搜一下最佳实践`
- `帮我看看怎么做`

AI 必须先改写成：

- 主题
- 目标问题
- 来源优先级
- 排除项
- 输出结构

### 3.2 先找权威来源，再找细节

AI 必须优先搜索：

1. 官方文档
2. 官方 help / best practices
3. 主作者 / 主维护者文章
4. 权威技术组织资料

AI 不应先从教程站、聚合站、二手总结开始。

### 3.3 先找结论，再找解释

AI 搜资料时，优先找：

- 明确边界结论
- 明确最佳实践
- 明确约束
- 明确推荐写法

AI 不应先收集大段泛介绍。

### 3.4 结果太泛时，必须继续缩小

当第一轮结果偏泛时，AI 必须主动做第二轮更窄检索。

收窄方式包括：

- 从大主题缩小到子问题
- 从泛问题缩小到边界问题
- 从资料搜索缩小到官方来源搜索
- 从“怎么做”缩小到“为什么这样做 / 哪条规则这样要求”

### 3.5 先查本地证据，再决定是否扩大到网络

对 `Isolation` 仓库自己的任务，AI 默认按下面顺序取证：

1. `AGENTS.md` 与 `docs/约束/*.md`
2. `C:\Users\shan\Downloads\api开源教程学习\` 里的目标参考仓文件
3. `C:\Users\shan\Downloads\rules学习\` 里的目标规则样本
4. 只有本地证据不足、事实明显时效敏感、或用户明确要求网络搜索时，才扩大到网络

这类任务包括：

- DDD / 模块 / 类设计判断
- `AGENTS.md`、提示词、约束文档措辞优化
- docs-as-code 守护、代码/文档同步、测试锚点设计

## 4. AI 在搜索前必须补齐的 8 个元素

每次网络搜索前，AI 必须先补齐下面 8 项：

1. **主题**
2. **目标问题**
3. **对象范围**
4. **来源优先级**
5. **排除项**
6. **时间要求**
7. **输出形式**
8. **落地动作**

如果原始用户请求没写全，AI 也要先在内部补齐，再发起搜索。

## 5. AI 的默认检索改写规则

### 5.1 原始问题改写模板

AI 应把用户请求优先改写成下面形状：

```text
网络搜索 <主题>，
目标是回答 <具体问题>，
优先使用 <官方文档 / 主作者 / 权威资料>，
排除 <二手总结 / 搬运文 / 过时版本 / 泛介绍>，
如果结果太泛，继续缩小问题，
最后输出：
1. 结论
2. 来源链接
3. 关键事实
4. 与当前任务直接相关的部分
```

### 5.2 当用户真正想要的是“某种信息”时

AI 必须显式加上：

```text
我要的不是泛介绍，而是 <目标信息>
```

可替换成：

- 我要的是官方结论
- 我要的是边界规则
- 我要的是最佳实践
- 我要的是设计建议
- 我要的是和当前代码直接相关的判断依据

### 5.3 当用户要求落地到仓库时

AI 必须显式加上：

```text
最后整理成适合本仓库使用的结论 / Markdown / 约束文档
```

### 5.4 当任务是仓库内设计 / 提示词 / 约束文档时

AI 必须显式先做本地证据改写：

```text
先检查 `AGENTS.md`、`docs/约束/*.md`、`C:\Users\shan\Downloads\api开源教程学习\`、
`C:\Users\shan\Downloads\rules学习\` 中与当前问题直接相关的最小证据集。
如果这些证据已经足够回答问题，就直接整理成适合本仓库使用的结论 / Markdown / 约束文档；
如果仍有缺口，再补官方 / 权威网络来源。
```

## 6. AI 的三段式检索流程

对 `Isolation` 仓库的 repo-specific 问题，先执行一个本地证据预检步骤，再进入三段式检索流程。

### 第零步：本地证据预检

AI 应先检查：

1. `AGENTS.md` 与相关 `docs/约束/*.md`
2. `C:\Users\shan\Downloads\api开源教程学习\` 下是否已有直接对应的参考文件
3. `C:\Users\shan\Downloads\rules学习\` 下是否已有更适合的 prompt / rule / AGENTS 样本

只有下面任一条件成立时，才扩大到网络：

- 本地证据没有覆盖目标问题；
- 目标事实明显依赖最新版本、最新规范或最新公告；
- 用户明确要求“网络搜索 / 查最新资料 / 找官方链接”。

### 第一步：找主来源

AI 应先搜：

```text
<主题> official docs
<主题> best practices official
<主题> author guide
```

目标：

- 锁定权威来源
- 不急着收集细节

### 第二步：从主来源中提关键结论

AI 应再搜：

```text
基于这些来源，继续查 <具体问题> 的明确结论
```

目标：

- 提取能直接回答问题的内容
- 排除泛百科描述

### 第三步：映射到当前任务

AI 应继续收窄：

```text
把这些结论映射到 <当前仓库 / 当前代码 / 当前架构问题>
```

目标：

- 保留和当前任务直接相关的内容
- 丢弃无关背景知识

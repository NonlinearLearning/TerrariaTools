# 上下文地图

## 1. 目的

本文记录当前仓库的上下文地图，以当前代码为准。

## 2. 当前地图

### 核心主链

`Workspaces -> Analysis -> Marking -> Propagation -> Decision -> Execution -> Rewrite -> Output`

当前对应的主链装配关系：

- `Application.Services.RewriteWorkflowAppService`
- `Logic.Workflow.RewriteWorkflowArtifactAssembler`
- `Logic.Workflow.RewriteWorkflow*Stage`
- `Domain.Decision / Execution / Output` 聚合与值对象

### 横向支撑

- `Rules`：规则编号、规则集、优先级、执行策略、作用域
- `Common`：共享值对象与聚合基类
- `Workflow.Events`：主链事件补齐、记录、回传

## 3. 当前上下文职责

### Workspaces

- 处理输入来源
- 管理路径、项目、引用、默认规则语义

### Analysis

- 提供分析快照与 Program Fact 语言
- `Analysis.Engine` 承担引擎内部实现

### Marking

- 识别规则命中目标
- 产出变更候选初始集

### Propagation

- 维护传播边界、传播轨迹、覆盖关系、传播事实引用

### Decision

- 对候选做风险解释和最终裁决

### Execution

- 维护计划、冲突、执行结果与完成条件

### Rewrite

- 维护重写产物：slice / shadow / closure / rewrite result

### Output

- 收集验证证据
- 生成审计结论

### Workflow.Events

- `WorkflowEventSequenceBuilder`：复用原生事件并补齐主链事件
- `IDomainEventRecorder / InMemoryDomainEventRecorder`：按关联标识记录、查询、清理事件
- `RewriteWorkflowStageResults.ToArtifacts(...)`：把阶段结果和事件回传为统一产物

## 4. 当前结论

1. 当前上下文地图已经足够支撑四层落位。
2. 当前地图重点在“协作关系清晰”，而不是继续拆更多上下文名词。
3. 当前最需要补的是自动化守护，不是再造一份更复杂的图。
4. `Workflow.Events` 当前已经从隐式收尾逻辑收敛成显式支撑上下文。

# 领域事件与上下文协作落地执行文档

## 1. 目的

本文记录当前仓库领域事件与阶段协作的落地现状。

重点回答：

- 事件从哪里产生

## 2. 当前事件主线

当前事件主线围绕 `RewriteWorkflow` 展开，分为：

1. Analysis
2. Marking
3. Propagation
4. Decision
5. Execution
6. Verification / Audit

## 4. 当前协作方式

当前协作已经不是“Application 手工拼一串事件”，而是：

- 聚合先产生原生事件
- `Logic.*.Events.*Publisher` 与 `*SequenceBuilder` 负责补齐阶段发布序列
- `WorkflowEventSequenceBuilder` 优先复用上游已有原生事件
- `RewriteWorkflowEventStage` 负责调用 `WorkflowEventSequenceBuilder` 与 `IDomainEventRecorder` 完成缺失事件补录和去重回传
- `InMemoryDomainEventRecorder` 负责按关联标识清理、记录、查询和去重辅助

## 6. 当前结论

1. 原生聚合事件已经成为主线。
2. 阶段发布器当前主要承担“拼接发布上下文”和“复用现有事件”的职责。
3. 下一步更值钱的方向是继续把稳定阶段边界转成 ArchitectureTests 和回归测试，而不是再造一层事件总线。
4. 当前事件主链的收尾阶段已经形成 `EventStage -> WorkflowEventSequenceBuilder -> DomainEventRecorder -> StageResults.ToArtifacts(...)` 的稳定闭环。

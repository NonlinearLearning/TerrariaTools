# 架构优化批次一执行计划

## 目标

围绕当前架构审查结论，先完成一批能形成闭环验证的最小改动，优先收敛应用层纯度与边界守护。

## 本批次合并执行的小任务

### 任务 1：纯化主链 `Application.Contracts`

合并范围：

- `Workflow`
- `Decision`
- `Propagation`
- `Execution`
- `Workspaces`
- `Marking`

执行目标：

- 把主链契约中直接暴露的 `Domain.*` 枚举和值对象替换为 contract enum / contract record / primitive fields。
- 把领域语义转换集中到 `Application.Mappers`。

### 任务 2：瘦身 `RewriteWorkflowAppService`

合并范围：

- 下沉 `BuildMarkingCandidates`
- 去掉应用服务中的工作流规则构造点

执行目标：

- 让 `RewriteWorkflowAppService` 更接近“取输入、调能力、回结果”。
- 将工作流标记候选准备逻辑收口到 `Logic.Workflow` 中的明确能力。

### 任务 3：补 `ArchitectureTests` guard rail

合并范围：

- 契约纯度守护
- 应用服务低层构造守护

执行目标：

- 为本批次新收敛的边界加自动化约束。
- 防止 `Application.Contracts` 和 `RewriteWorkflowAppService` 回退。

## 明确不在本批次完成的目标

- `Logic.Workflow` 的进一步分裂：计划编译 / 执行 / 证据 / 报告 / 事件序列 五块内部继续拆分
- `Analysis / Output / Rewrite.Artifacts` 契约纯化
- 更深入的 `Domain Service` 富化与 `Decision / Propagation / Execution` 规则回收

## 验证标准

1. 主链契约文件不再直接 `using Domain.Decision / Domain.Execution / Domain.Propagation / Domain.Workspaces / Domain.Rules`
2. `RewriteWorkflowAppService` 不再持有 `BuildMarkingCandidates`
3. `ArchitectureTests` 能对上述边界提供 guard rail
4. 相关构建和架构验证通过
